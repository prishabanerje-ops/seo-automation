# Screaming Frog CLI Limitations

## Overview

Screaming Frog SEO Spider (version 23.3) supports headless operation via CLI.
However, many settings that are configurable in the GUI **cannot be set via CLI flags**.
These must be saved to a binary `.seospider` config file using the SF GUI,
then loaded per crawl.

## Settings Available as CLI Flags

| Setting | CLI Flag |
|---|---|
| Start URL (spider mode) | `--crawl <url>` |
| URL list (list mode) | `--crawl-list <file>` |
| Sitemap (sitemap mode) | `--crawl-sitemap <url>` |
| Output folder | `--output-folder <path>` |
| Export tabs | `--export-tabs <tab:filter,...>` |
| Export format | `--export-format csv` |
| Save crawl project | `--save-crawl` |
| Overwrite output | `--overwrite` |
| Headless mode | `--headless` |

## Settings NOT Available as CLI Flags (GUI-Only)

The following settings exist in the CrawlRunner UI but are currently **not applied** to crawls because SF CLI does not support them as flags:

| Setting | Why It Cannot Be Set |
|---|---|
| Max crawl depth | No `--max-depth` flag (causes FATAL error if added) |
| Crawl delay (ms) | No `--crawl-delay` flag |
| User agent | No `--user-agent` flag |
| JavaScript rendering mode | No `--rendering` flag |
| Max URLs | No `--max-urls` flag |
| Thread count | No `--threads` flag |

These settings are displayed in the UI with a warning tooltip indicating they require a pre-saved SF binary config.

## How to Apply These Settings

### Method 1: Pre-saved SF Config (Recommended)

1. Open Screaming Frog GUI
2. Configure the desired settings (depth, UA, rendering, etc.)
3. Go to File → Save Configuration
4. Save the `.seospiderconfig` file
5. Pass it to the CLI using `--config /path/to/config.seospiderconfig`

**Note:** SF saves config in binary Java serialization format — the XML config generated by `sf-config.service.js` is NOT compatible with the `--config` flag.

### Method 2: SF Default Preferences

Settings saved via File → Preferences in the SF GUI are applied to all headless crawls automatically (stored in `~/.ScreamingFrogSEOSpider/prefs/`).

## Impact on Current Crawls

- **Spider mode** on large sites (50k+ URLs) will crawl without limits and run indefinitely
- **Recommended workaround**: Use list mode (feed URLs from GSC export) or sitemap mode for large sites
- Spider mode is suitable for small sites (< 5,000 URLs) where a full crawl is feasible

## SF License

The SF license is stored at `~/.ScreamingFrogSEOSpider/licence.txt` and is read automatically by the CLI. There is no `--license` CLI flag. The free tier limits crawls to 500 URLs.
